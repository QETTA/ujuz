## 1. 목적
이 문서는 UjuZ 핵심 UX(어린이집 탐색, 시설 비교, TO 알림, 입학상담) 전반의 일관성을 유지하기 위한 기준이다.  
디자인/기획/개발/QA가 같은 단어와 같은 상태 정의를 사용하도록 통일한다.

## 2. 제품 컨텍스트
- 사용자: 만 0~7세 자녀를 둔 보호자(정보 과부하 + 시간 압박 + 의사결정 불안)
- 핵심 행동: `검색/지도 탐색 -> 시설 저장/비교 -> TO 알림 설정 -> 상담 구매/리포트 확인`
- 제품 약속: "빠르게, 근거 있게, 불안을 줄이며 결정하도록 돕는다"

## 3. 디자인 원칙
1. 가치 먼저 보여준다: 첫 화면에서 "내 주변 + TO 기회 + 다음 행동"을 즉시 제시한다.
2. 불안을 키우지 않는다: 공포 유도 문구 대신 "현재 상황 + 가능한 선택지"를 함께 제시한다.
3. 권한은 필요 시점에만 요청한다: 기능 진입 직전(Just-in-time) 요청, 거절 시 대체 경로를 항상 제공한다.
4. 비교는 짧고 선명하게 만든다: 3~5개 핵심 비교축(거리, 비용, 운영시간, 평점/리뷰 신뢰도, TO 가능성)만 우선 노출한다.
5. 근거를 분리해 보여준다: 요약(결론)과 근거(데이터 출처/불확실성)를 분리해 이해 부담을 낮춘다.
6. 다음 행동을 명확히 둔다: 각 화면에 Primary CTA는 1개만 둔다.
7. 상태 전환을 예측 가능하게 만든다: loading/empty/error/success 패턴을 화면 간 동일하게 유지한다.
8. 민감정보는 최소 수집한다: 아이 정보는 "목적-보관기간-삭제경로"를 함께 설명한다.

## 4. Tone/Copy 규칙
### 4.1 기본 톤
- 공감은 하되 과장하지 않는다.
- 단정 대신 확률/조건을 명시한다.
- 전문용어보다 생활어를 우선한다.

### 4.2 문장 규칙
1. 제목은 16자 내외, 한 문장 한 메시지.
2. 버튼은 동사형으로 끝낸다. 예: `알림 받기`, `비교에 추가`, `상담 예약`.
3. 에러 문구는 원인 + 대안 순서로 쓴다.
4. 숫자는 맥락과 함께 표기한다. 예: `평균 답변 24시간`, `예상 대기 3~5개월`.

### 4.3 금지 표현
- 금지: "지금 안 하면 늦어요", "무조건", "100% 입소"
- 허용: "현재 기준", "가능성이 높은 편", "다음 선택"

### 4.4 화면별 카피 템플릿
- TO 알림 프리프롬프트: `놓치기 쉬운 TO 변동을 알려드릴게요`
- 권한 거절 후 폴백: `권한 없이도 알림센터에서 새 소식을 확인할 수 있어요`
- 상담 CTA: `우리 아이 조건으로 상담받기`

## 5. 상태 네이밍 표준
### 5.1 기본 형식
`[Phase]/[도메인]/[화면]/[상태]`

예시:
- `P6/알림/목록/default`
- `P6/알림/목록/empty_first_use`
- `P6/알림/구독생성/permission_prompt`
- `P7/상담/리포트/ready`

### 5.2 상태 값 표준
- 기본: `default`
- 로딩: `loading`, `skeleton`
- 비어있음: `empty_first_use`, `empty_no_results`, `empty_no_alerts`
- 오류: `error_network`, `error_permission`, `error_server`
- 성공/피드백: `success_toast`, `saved`, `completed`
- 처리중: `processing`, `writing`

### 5.3 작명 규칙
1. `snake_case` 사용 (`empty_first_use`).
2. 의미 없는 이름 금지 (`none`, `etc`, `state1`).
3. 상태는 상호배타적으로 설계한다(한 시점에 대표 상태 1개).
4. 빈 상태는 "첫 사용"과 "검색/조건 결과 없음"을 분리한다.

## 6. 적용 예시
### 알림 목록 화면
- `default`: 최근 알림 리스트 + unread 배지
- `loading`: 스켈레톤 3개 노출
- `empty_first_use`: "아직 구독한 알림이 없어요" + `알림 만들기`
- `empty_no_alerts`: "지금은 새 소식이 없어요" + `조건 넓히기`
- `error_network`: 재시도 버튼 + 마지막 성공 시각

### 시설 상세 화면
- `default`: 핵심정보 + `비교에 추가` + `TO 알림 받기`
- `error_server`: 요약 카드 숨김, 기본 정보만 폴백 노출

## 7. 릴리즈 전 점검
- 모든 핵심 화면에 `loading/empty/error` 상태가 정의되어 있는가
- 권한 거절 시 대체 경로가 실제 동작하는가
- 카피가 불안을 과도하게 자극하지 않는가
- "근거/불확실성" 표기가 유지되는가
