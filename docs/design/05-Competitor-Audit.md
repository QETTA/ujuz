## 1. 목적과 범위
부모의 "어린이집 탐색 -> 비교 -> 알림 -> 의사결정/상담" 흐름을 기준으로 경쟁 서비스의 강약점을 정리한다.  
본 문서는 디자인 방향성 수립용 데스크 리서치 바인더이며, 앱 업데이트에 따라 주기적 검증이 필요하다.

## 2. 비교 대상
| 서비스 | 유형 | 주요 사용자 단계 |
|---|---|---|
| UjuZ | 직접 경쟁(입소 전 의사결정) | 탐색/비교/TO 알림/상담 |
| 아이사랑(정부) | 직접 경쟁(입소대기/공공정보) | 검색/대기신청/행정 확인 |
| 키즈노트 | 인접 경쟁(재원 후 소통) | 공지/알림장/결제 |
| 네이버지도 + 카페/블로그 | 대체재(조합형) | 검색/리뷰 탐색/상담 문의 |
| 지역 맘카페(네이버 카페 기반) | 대체재(커뮤니티 추천) | 후기 탐색/질문/문의 |

## 3. 퍼널 비교 요약
평가 기준: `강함`, `보통`, `약함`, `없음`

| 서비스 | 검색 | 지도/거리 | TO/대기 알림 | 결제 | 상담/의사결정 지원 |
|---|---|---|---|---|---|
| UjuZ | 강함 | 강함 | 강함(목표) | 보통(상담 결제 중심) | 강함(PDF 리포트/상담) |
| 아이사랑(정부) | 보통 | 약함 | 보통(행정 중심) | 없음 | 약함 |
| 키즈노트 | 약함(입소 전 탐색 아님) | 없음 | 보통(재원 공지 알림) | 보통~강함(원비/활동비) | 약함 |
| 네이버지도+카페 | 강함 | 강함 | 약함(실시간 TO 어려움) | 없음 | 보통(후기 기반) |
| 지역 맘카페 | 보통 | 약함 | 약함 | 없음 | 보통(경험담 중심) |

## 4. 퍼널 단계별 인사이트
### 4.1 검색/지도
- 강점 경쟁자: `네이버지도+카페`
- 사용자 체감: 정보량은 많지만 신뢰도/최신성 판단이 어렵다.
- UjuZ 기회: "검색 결과 자체"보다 "비교가 바로 되는 구조"를 차별점으로 강화.

### 4.2 TO/대기 알림
- 강점 경쟁자: `아이사랑`(공식 대기 맥락), `키즈노트`(알림 습관)
- 공백: 실시간 TO 변동을 개인 조건으로 필터링해 알려주는 경험은 약함.
- UjuZ 기회: 권한 허용 유도보다 "거절해도 앱 내 알림센터" 폴백이 핵심.

### 4.3 결제
- 강점 경쟁자: `키즈노트`(재원 후 정기 결제 경험)
- 공백: 입소 전 의사결정 단계에서 "구매할 만한 정보 상품" 포지셔닝이 약함.
- UjuZ 기회: 상담 결제는 "불안 감소 + 실행 계획"을 구체적으로 보여줘야 전환된다.

### 4.4 상담/의사결정
- 강점 경쟁자: 커뮤니티(경험담), 블로그(상세 후기)
- 공백: 근거 기반 + 개인화 + 실행 체크리스트를 한 번에 제공하는 구조가 드물다.
- UjuZ 기회: `요약 3줄 -> 체크리스트 -> 근거 링크 -> 상담 CTA` 패턴 고정.

## 5. UjuZ 기회와 우선순위
1. `지도 + 비교 + TO 알림`을 한 플로우로 묶어 "다음 행동"을 끊기지 않게 설계
2. 알림 이후 행동 설계 강화: `알림 클릭 -> 비교 업데이트 -> 상담/저장` 단축
3. 상담 구매 불안 해소: 샘플 PDF, 예상 소요시간, 환불/노쇼 정책 선노출
4. 공공정보/커뮤니티 후기와 충돌 시 "근거와 불확실성" 표시로 신뢰 확보

## 6. 후속 리서치 액션
1. 실제 사용자 5명에게 경쟁 서비스 사용 로그 인터뷰(최근 2주 행동 회상)
2. "정보 신뢰 판단 기준" 질문으로 UjuZ Evidence UI 우선순위 확정
3. 알림 빈도/시간대 선호를 구간별(즉시/요약/조용한시간)로 정량 수집
