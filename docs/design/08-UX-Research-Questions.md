## 1. 목적
아래 12문항은 UjuZ 인터뷰에서 바로 읽고 사용할 수 있도록 작성한 반구조화 질문 세트다.  
각 문항은 제품 의사결정(권한, 알림, 상담, 결제, 리텐션)으로 직접 연결된다.

## 2. 인터뷰 핵심 질문 12개
| 번호 | 인터뷰 질문(그대로 읽기) | 후속 프로브 | 의사결정 연결 |
|---:|---|---|---|
| 1 | "어떤 순간에 위치 권한을 요청받으면 가장 덜 부담스럽나요?" | 첫 실행/검색 직후/거리순 정렬 시점 비교 | 위치 권한 요청 타이밍 |
| 2 | "위치 권한을 거절했을 때도 계속 쓸 수 있으려면 어떤 대안이 필요할까요?" | 지역 직접 선택, 최근 지역 저장 선호 | 거절 폴백 설계 |
| 3 | "TO 알림은 얼마나 자주 받는 게 적당하다고 느끼세요?" | 즉시/하루 1회/주간 요약 선호 이유 | 푸시 빈도 기본값 |
| 4 | "알림이 오는 시간대에서 피하고 싶은 시간이 있나요?" | 조용한 시간대, 주말 알림 허용 여부 | 알림 시간 정책 |
| 5 | "TO 알림을 받으면 실제로 어떤 행동을 하시나요?" | 상세 보기/배우자 공유/전화 문의 순서 | 알림 후 행동 퍼널 |
| 6 | "입학상담 상품을 결제하려고 할 때 가장 불안한 점은 무엇인가요?" | 가격, 품질, 환불, 전문가 신뢰 | 상담 구매 불안 해소 |
| 7 | "상담 후 받는 PDF 리포트에서 꼭 있어야 하는 정보는 무엇인가요?" | 실행 체크리스트, 일정표, 근거 링크 우선순위 | PDF 리포트 구조 |
| 8 | "결과를 볼 때 체크리스트형과 설명형 중 어떤 형식이 더 도움 되나요?" | 상황별 선호 변화(급할 때/여유 있을 때) | 결과 UI 포맷 |
| 9 | "아이 정보(성향, 건강, 발달)를 입력할 때 어느 수준부터 민감하다고 느끼세요?" | 항목별 거부감, 저장 기간 기대치 | 개인정보 최소수집 범위 |
| 10 | "보통 몇 개 시설까지 저장/비교하면 결정하기 좋다고 느끼세요?" | 3개/5개/10개 이상, 관리 방식 | 저장/비교 UX 한계치 |
| 11 | "이 앱을 계속 쓰게 만드는 핵심 이유는 무엇일까요?" | 알림 정확도, 정보 신뢰, 상담 품질 중 우선순위 | 핵심 리텐션 요인 |
| 12 | "상담 결제를 하겠다는 마음은 보통 어떤 시점에 생기나요?" | TO 알림 직후/방문 전/배우자 합의 후 | 결제 의향 시점 설계 |

## 3. 진행 팁
1. 질문은 순서대로 읽되, 한 주제에서 깊은 사례가 나오면 2~3분 추가 탐색
2. "왜 그렇게 느꼈는지"를 최소 2회 반복 확인
3. 답변은 반드시 실제 행동 사례(최근 1개월)로 재확인
