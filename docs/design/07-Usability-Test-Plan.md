## 1. 테스트 목적
UjuZ의 핵심 퍼널 `검색/지도 -> 시설 비교 -> TO 알림 설정 -> 상담 결제 의사결정`에서 이탈 원인과 신뢰 저해 요소를 찾는다.

## 2. 검증 가설
1. 사용자는 3분 내에 "내 주변 시설 2개 비교"를 완료할 수 있다.
2. 사용자는 TO 알림 권한 요청의 타이밍을 자연스럽게 받아들인다.
3. 알림 이후 다음 행동(저장/비교/상담)이 명확하면 이탈이 줄어든다.
4. 상담 결제 전 불안 요소(가격/품질/환불)가 선노출되면 구매 의향이 상승한다.

## 3. 참가자 조건(5명)
- 보호자 5명(자녀 0~7세)
- 지역 다양성: 대도시 3, 중소도시 2
- 상태 분포: 현재 대기 중 3, 입소 검토 중 2
- 제외: 현직 보육교사/어린이집 운영자(전문가 편향 방지)

## 4. 세션 구성(60분)
1. 오프닝/동의 (5분)
2. 배경 질문 (8분)
3. 과업 수행 (35분)
4. 사후 인터뷰 (10분)
5. 마무리 (2분)

## 5. 과업 시나리오
### Task 1. 내 주변 시설 찾기
- 지시문: "집 근처에서 통학이 쉬운 어린이집을 찾아주세요."
- 성공 기준: 지도에서 후보 3개 이상 확인 + 1개 상세 진입

### Task 2. 시설 2개 비교/저장
- 지시문: "마음에 드는 두 시설을 비교하고 저장해 주세요."
- 성공 기준: 비교 화면 진입 + 저장 완료

### Task 3. TO 알림 설정
- 지시문: "가장 원하는 시설의 TO 알림을 켜주세요."
- 성공 기준: 알림 구독 생성 + 권한 의사결정 완료

### Task 4. 알림 수신 후 행동 선택
- 지시문: "TO 알림을 받았다고 가정하고 다음 행동을 해주세요."
- 성공 기준: 상세 재확인 -> 비교 업데이트 또는 상담 CTA 진입

### Task 5. 상담 상품 검토/결제 의향 판단
- 지시문: "입학상담 상품을 보고 구매 여부를 결정해 주세요."
- 성공 기준: 구매/보류 결정 + 이유 2개 이상 언급

## 6. 진행 스크립트
### 6.1 오프닝
- "오늘은 앱 사용성을 확인하는 테스트이며, 참가자님의 능력을 평가하지 않습니다."
- "불편하거나 헷갈리는 지점을 소리 내어 말씀해 주세요."

### 6.2 과업 중 공통 멘트
- 시작: "지금부터 평소처럼 사용해 주세요."
- 정지 시: "지금 무엇을 기대하고 계신가요?"
- 실패 시: "어떤 정보가 있었으면 계속 진행하기 쉬웠을까요?"

### 6.3 사후 질문
- "가장 신뢰가 갔던 화면은 어디였나요?"
- "결제를 망설이게 만든 요소는 무엇이었나요?"
- "알림 빈도/시간대 설정은 본인 스타일과 맞았나요?"

## 7. 측정 지표
- 과업 성공률(%)
- 과업 완료 시간(초)
- 치명적 오류 개수(진행 불가)
- 도움 요청 횟수
- SEQ(7점 척도), 신뢰도 점수(5점 척도), 결제 의향(5점 척도)
