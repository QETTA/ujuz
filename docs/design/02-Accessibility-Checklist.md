## 1. 목적과 범위
UjuZ 핵심 화면(지도 탐색, 시설 상세/비교, TO 알림, 상담/결제 플로우)의 접근성을 WCAG 2.2 기준으로 점검한다.  
대상 플랫폼은 웹(Next.js) + 모바일(Expo iOS/Android)이다.

## 2. WCAG 2.2 핵심 체크리스트
### 2.1 Perceivable(인식 가능)
- [ ] 텍스트 대비 4.5:1 이상, 큰 텍스트 3:1 이상
- [ ] 아이콘/차트/배지 등 비텍스트 UI 3:1 이상
- [ ] 색만으로 상태 전달하지 않음(아이콘/텍스트 동시 제공)
- [ ] 200% 확대(또는 OS 글자 크기 확대) 시 정보 손실 없음
- [ ] 이미지/아이콘 버튼에 대체 텍스트 제공

### 2.2 Operable(조작 가능)
- [ ] 주요 탭/버튼 터치 영역 충분(iOS 44pt+, Android 48dp+ 권장)
- [ ] 키보드/스위치 접근(웹 기준 Tab 이동, 모바일 보조입력) 가능
- [ ] 포커스 이동 순서가 시각적 순서와 일치
- [ ] 포커스가 가려지지 않음(WCAG 2.2 Focus Not Obscured)
- [ ] 드래그 전용 인터랙션에는 탭 대체 경로 제공(WCAG 2.2 Dragging Movements)
- [ ] 시간 제한/자동 갱신이 있으면 일시정지/재시도 수단 제공

### 2.3 Understandable(이해 가능)
- [ ] 오류 메시지는 원인 + 해결 방법을 함께 제공
- [ ] 동일 기능의 라벨/위치 일관성 유지
- [ ] 입력값 재요청 시 이미 입력한 정보 재사용(중복 입력 최소화)
- [ ] 경고/확인 모달의 결과가 예측 가능(취소/뒤로가기 명확)

### 2.4 Robust(견고성)
- [ ] 시맨틱 역할(role)과 접근성 라벨이 일치
- [ ] 상태 변경(로딩/성공/오류)이 스크린리더에 전달됨
- [ ] 커스텀 컴포넌트가 기본 제스처/접근성 API를 깨지 않음

## 3. iOS 체크리스트
- [ ] VoiceOver 읽기 순서가 시각 순서와 동일
- [ ] Dynamic Type `XS~AX5`에서 텍스트 잘림/겹침 없음
- [ ] `Reduce Motion` 활성화 시 과한 애니메이션 비활성/완화
- [ ] 권한 프롬프트(알림/위치) 전 설명 화면(프리프롬프트) 제공
- [ ] 권한 거절 후 앱 내 대체 흐름(수동 지역 선택, 앱 내 알림센터) 제공
- [ ] 하단 탭/상단 네비게이션이 Safe Area 침범 없음

## 4. Android 체크리스트
- [ ] TalkBack 라벨/힌트가 버튼 목적과 일치
- [ ] 시스템 폰트 스케일 1.3/1.5/2.0에서 레이아웃 붕괴 없음
- [ ] 48dp 미만 탭 타깃 없음
- [ ] 뒤로가기(제스처/하드웨어) 동작이 예측 가능
- [ ] 권한 거절 시 재시도/설정 이동 경로가 명확
- [ ] 대비 강화/다크 테마에서도 정보 손실 없음

## 5. 테스트 실행 순서
1. 디자인 단계: 피그마에서 대비/타깃 크기/상태별 카피 점검
2. 개발 단계: 컴포넌트 단위 접근성 라벨/포커스 점검
3. QA 단계: iOS VoiceOver + Android TalkBack 실제 시나리오 테스트
4. 회귀 테스트: TO 알림 설정, 시설 비교, 상담 구매 흐름 재검증

## 6. 릴리즈 게이트
아래 6개가 모두 통과해야 배포 가능:
1. 권한 거절 시 핵심 기능 사용 가능(검색/비교/앱 내 알림 확인)
2. 스크린리더로 핵심 과업 5개 수행 가능
3. 글자 크기 확대로도 CTA 접근 가능
4. 오류 메시지가 해결 행동을 제공
5. 색각 이상 사용자도 상태 인식 가능
6. 결제/상담 플로우에서 입력 오류 복구 가능
